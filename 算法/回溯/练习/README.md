# 回溯练习顺序建议

建议按下面顺序练习，逐步掌握“状态表示 -> 约束判断 -> 尝试/回退 -> 去重/剪枝”。

1. 回溯算法_练习.py
- 先做 `find_nodes_with_value_7`：只练 DFS 遍历与条件收集。
- 再做 `find_paths_to_value_7`：加入路径栈 `path`，体会“先加入、后弹出”。
- 然后做 `find_paths_to_value_7_without_3`：在递归入口加剪枝。
- 最后补全通用 `backtrack`：抽象回溯模板。

2. 全排列问题_练习.py
- 先做 `permutations_i`：用 `selected` 防止重复选同一元素。
- 再做 `permutations_ii`：加“同层去重”集合 `duplicated`。

3. 子集和问题_练习.py
- 先做 `subset_sum_i_naive`：理解基本回溯与超目标剪枝。
- 再做 `subset_sum_i`：通过排序 + `start` 指针去重，并利用 `break` 提前终止。

4. N皇后问题_练习.py
- 综合题：同时维护列、主对角线、次对角线约束。
- 重点检查“放置皇后”和“撤销皇后”是否完全对称。

## 建议自检清单

- 每次递归进入前后，状态是否严格对称（有加必有减）。
- 记录解时是否做了“拷贝”而不是直接引用原列表。
- 剪枝条件是否放在足够靠前的位置。
- 去重是“同层去重”还是“全局去重”，是否符合题意。

## 小提示

- 每补完一个函数，先用一组最小样例手算，再运行验证。
- 若结果为空或重复，优先检查：
  - 是否漏了回退。
  - 是否误用了递归参数（如 `start`、`remaining`、`row`）。
