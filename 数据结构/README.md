# 数据结构学习顺序

这个目录下面的内容建议按“先基础、后综合、再图论”的顺序练习。

## 推荐练习顺序

1. 数组与链表
- 先把数组、列表、链表这些最基础的数据容器练熟。
- 重点理解：顺序访问、插入删除、双指针、指针/索引移动。

2. 栈与队列
- 在线性结构的基础上，练习先进后出和先进先出两种约束。
- 重点理解：括号匹配、单调结构、BFS 相关的队列使用。

3. 树
- 先练二叉树的遍历，再练二叉搜索树和 AVL 树。
- 重点理解：递归 DFS、层序遍历、BST 性质、平衡与旋转。

4. 堆
- 在树的基础上继续练完全二叉树和优先级处理。
- 重点理解：上浮、下沉、建堆、Top-k 问题。

5. 哈希表
- 练习用“空间换时间”的思路解决查找问题。
- 重点理解：哈希函数、冲突处理、开放寻址、链式地址。

6. 图
- 放在最后练，先掌握邻接表/邻接矩阵，再做 BFS 和 DFS。
- 重点理解：图的建模、遍历、连通性与路径搜索。

## 建议

1. 每个文件先只看函数签名和步骤提示，自己独立完成。
2. 同一类结构练完后，回头对照原实现复盘边界条件和写法差异。
3. 如果时间有限，优先完成“数组与链表、栈与队列、树”这三组，再继续往后走。
